Current Server API Endpoints (Paths)

Below is a description of the currently implemented API endpoints that the server exposes. These paths allow various client applications (web, mobile, etc.) to interact with our backend services.

  1. GET /api/status

    Description: Provides a simple health check for the server. Returns the server's current status and possibly a timestamp, indicating that the server is up and running.

    Response Example:

    {
    "status": "operational",
    "message": "Network Drivers server is online!",
    "timestamp": "2023-10-27T10:30:00Z"
}

  2. POST /api/users/register

    Description: Handles new user registration. Expects user credentials (e.g., username, email, password) in the request body to create a new user account.

    Request Body Example:

    {
    "username": "newuser",
    "email": "newuser@example.com",
    "password": "securePassword123"
}

    Response Example (Success):

    {
    "success": true,
    "message": "User registered successfully",
    "userId": "uuid-of-new-user"
}

  3. POST /api/users/login

    Description: Authenticates existing users. Expects username/email and password. On successful authentication, it typically returns an authentication token (e.g., JWT).

    Request Body Example:

    {
    "email": "existing@example.com",
    "password": "securePassword123"
}

    Response Example (Success):

    {
    "success": true,
    "message": "Login successful",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

  4. GET /api/data/fetch/:id (Requires Authentication)

    Description: Retrieves specific data by its ID. This endpoint requires user authentication (e.g., a valid JWT in the `Authorization` header). Replaces :id with the actual data identifier.

    Headers: Authorization: Bearer [your_token]

    Response Example:

    {
    "id": "data-item-123",
    "owner": "uuid-of-user",
    "content": "This is protected data.",
    "creationDate": "2023-10-26T15:00:00Z"
}

  5. POST /api/data/submit (Requires Authentication)

    Description: Allows authenticated users to submit new data to the server. The data structure will depend on the project's specific requirements.

    Headers: Authorization: Bearer [your_token]

    Request Body Example:

    {
    "dataType": "sensorReading",
    "value": 25.5,
    "unit": "Celcius",
    "timestamp": "2023-10-27T11:00:00Z"
}

    Response Example (Success):

    {
    "success": true,
    "message": "Data submitted successfully",
    "dataId": "uuid-of-new-data"
}

Key Packages and Technologies Used

Our server application leverages the following significant packages and technologies to achieve its functionality:

  • Express.js: A fast, unopinionated, minimalist web framework for Node.js, forming the backbone of our API.
  • Node.js: The JavaScript runtime environment that executes our server-side code.
  • Mongoose: An elegant MongoDB object modeling tool, used for interacting with our NoSQL database.
  • MongoDB: Our chosen NoSQL database for flexible and scalable data storage.
  • bcrypt.js: A library for hashing passwords, ensuring user credentials are stored securely.
  • jsonwebtoken (JWT): Used for implementing token-based authentication, allowing secure stateless user sessions.
  • cors: A Node.js package for providing a Connect/Express middleware that can be used to enable Cross-Origin Resource Sharing (CORS).
  • dotenv: Loads environment variables from a .env file, crucial for managing sensitive configuration data.
  • **Nodemon:** (Development dependency) Automatically restarts the node application when file changes are detected, speeding up development.

This list will be updated as our project progresses and new technologies or libraries are integrated.
